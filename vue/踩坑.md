## vue2.0踩坑
#### 1.导出的css路径，问题解决
vue-cli2版本中默认使用了ExtractTextPlugin插件，来合并各个vue文件的css。

build/utils.js:
```

if (options.extract) {
      return ExtractTextPlugin.extract({
        use: loaders,
        publicPath: '../../',         // 注意配置这一部分，根据目录结构自由调整
        fallback: 'vue-style-loader'
      })
    } else {
      return ['vue-style-loader'].concat(loaders)
    }
```
webpack.base.conf.js

```
plugins: [
    new ExtractTextPlugin({filename: "main.css", allChunks: true}), //抽离成一个单独的css
    new webpack.ProvidePlugin({
      $: "jquery",
      jQuery: "jquery",
      "windows.jQuery": "jquery"
    })
  ],
  
  ...
  {
    test: /\.vue$/,
    loader: 'vue-loader',
    options: {loaders:{
      css: ExtractTextPlugin.extract({
          use: 'css-loader',
          fallback: 'vue-style-loader'
      })
    }}
  },
```

#### 7.动态修改路由参数的query值


vue-cli中已经默认安装了webpack-merge
```

import merge from 'webpack-merge'；
 
修改原有参数        
this.$router.push({
    query:merge(this.$route.query,{'maxPrice':'630'})
})
 
新增一个参数：
this.$router.push({
    query:merge(this.$route.query,{'addParams':'我是新增的一个参数，哈哈哈哈'})
})
 
替换所有参数：
 this.$router.push({
    query:merge({},{'maxPrice':'630'})

```

#### 12.项目优化
##### 1.按需引入第三方库，不要把一些较大的库直接全部引入（打包时webpack会根据引用去打包依赖的内容）
```
// 正确用法
import isPlainObject from 'lodash/isPlainObject'

//错误用法
import { isPlainObject } from 'lodash'
```

##### 2.将逻辑代码和第三方库分离(DllPlugin)，加快构建速度
第一部：build文件夹下新建webpack.dll.conf
```
var path = require("path");
var webpack = require("webpack");
const UglifyJsPlugin = require('uglifyjs-webpack-plugin')

module.exports = {
  // 你想要打包的模块的数组
  entry: {
    vendor: [
      'vue/dist/vue.esm.js',
      'vuex',
      'axios',
      'vue-router',
      'element-ui',
      'xlsx',
      'echarts',
      'jquery'
    ]
  },
  output: {
    path: path.join(__dirname, '../static/js'),
    filename: 'dll.[name]_[hash:6].js',
    library: '[name]_[hash:6]' // 这里的library和下面的name必须对应
  },
  plugins: [
    new webpack.DllPlugin({
      path: path.join(__dirname, '../', '[name]-manifest.json'),
      name: '[name]_[hash:6]',
      context: __dirname
    }),
    new UglifyJsPlugin({
      uglifyOptions: {
        compress: {
          warnings: false,
          drop_console: true, // 移除代码中的console.log
          pure_funcs: ['console.log']
        }
      },
      sourceMap: false,
      parallel: true
    })
  ]
};
```

第二步：编辑build/webpack.base.conf，添加
```
plugins: [
  new webpack.DllReferencePlugin({
    context: __dirname,
    manifest: require('../vendor-manifest.json')
  })
],
```

第三步：在`package.json`中添加命令
```
"dll": "webpack --config build/webpack.dll.conf.js"
```

第四步：执行`npm run dll`
```
static/js 中生成vendor.dll_xx.js
根目录生成vendor-manifest.json，用来指引第三方库文件
```

第五步：在`index.html`中引入js,
```
<script src="/static/js/dll.vendor_ec4a9a.js"></script>
```



##### 3.懒加载路由
首先安装`babel-plugin-dynamic-import-node`
```
cnpm install babel-plugin-dynamic-import-node --save-dev
```

修改`.babelrc`文件,添加插件`dynamic-import-node`
```
{
  "presets": [
    ...
  ],
  "plugins": ["dynamic-import-node"]
}
```

修改路由,使用特殊的注释语法来提供 chunk name(Webpack > 2.4)
```
const Foo = () => import(/* webpackChunkName: "group-foo" */ './Foo.vue')
```
